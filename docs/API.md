# API Documentation\n\n## Overview\n\nThe eRechnung PDF/A-3 XML Extractor API provides endpoints for processing German electronic invoices embedded in PDF/A-3 documents.\n\n## Authentication\n\nCurrently, the API does not require authentication. This may change in future versions.\n\n## Rate Limits\n\nNo rate limits are currently enforced, but consider implementing them for production use.\n\n## Error Codes\n\n| Status Code | Description |\n|-------------|-------------|\n| 200 | Success |\n| 400 | Bad Request - Invalid file or format |\n| 500 | Internal Server Error |\n\n## Response Format\n\nAll responses are in JSON format with appropriate HTTP status codes.\n\n### Success Response Structure\n```json\n{\n  \"file status\": \"Success\",\n  \"embedded files\": \"factur-x.xml\",\n  \"xml_content\": \"<xml content>\",\n  \"xml_filename\": \"factur-x.xml\"\n}\n```\n\n### Error Response Structure\n```json\n{\n  \"file status\": \"Error description\",\n  \"embedded files\": \"list of files found (optional)\"\n}\n```\n
